# GHAES Session Mode — Avança Hospital Canindé Site

At the beginning of each AI-assisted development session, load this file once and apply it for the entire session.

This repository follows GHAES — Global Health AI Engineering Standard.

Reference:
https://github.com/erickgomesal/ghaes

## Session Rule

Use the repository instructions from:

- AGENTS.md
- CLAUDE.md
- CODEX.md
- SITE-STANDARD.md
- PUBLICATION-STANDARD.md

Do not re-read all instruction files before every prompt unless the task requires it.

For normal tasks, apply the already loaded GHAES rules from the session context.

## Re-read Instructions Only When

Re-read the repository standards when the task touches:

- institutional identity;
- public content;
- SEO;
- Open Graph;
- favicon;
- Netlify deployment;
- robots.txt;
- sitemap.xml;
- internal navigation;
- public project architecture;
- legal, political or funding-related text;
- unclear or conflicting instructions.

## Mandatory Rules

- Make surgical, minimal and traceable changes.
- Do not commit without explicit authorization.
- Do not push without explicit authorization.
- Do not change institutional architecture without approval.
- Do not invent public claims, numbers, laws or funding sources.
- Do not remove technical credibility.
- Do not expose sensitive or private data.
- Preserve the public identity of Avança Hospital Canindé.
- Preserve GSI HealthTech as the institutional ecosystem.
- Preserve GSI ONE as the platform/system.
- Preserve Avança Hospital as the implementation program/case.

## Required Final Report

Every task must end with:

1. Summary
2. Files changed
3. Public content impact
4. SEO/publication impact
5. Visual/UI impact
6. Validation performed
7. Risks
8. Pending decisions
9. Commit/push status
