# SITE-STANDARD.md — Avança Hospital Canindé Public Site Standard

This document defines local rules for the Avança Hospital Canindé public website.

This repository follows GHAES — Global Health AI Engineering Standard.

Reference:
https://github.com/erickgomesal/ghaes

## 1. Site Role

This website is the public institutional presentation of the Avança Hospital Canindé project.

It must communicate:

- the project vision;
- technical credibility;
- institutional purpose;
- implementation phases;
- project volumes;
- funding/capture strategy;
- indicators;
- public value.

## 2. Protected Architecture

Official architecture:

- GSI HealthTech = ecosystem and institutional brand.
- GSI ONE = platform/system.
- Avança Hospital = implementation program/case.
- Avança Hospital Canindé = first public implementation/case.

Do not rename or reinterpret this architecture without explicit approval.

## 3. Public Content Rules

The site must not contain:

- invented public data;
- unsupported legal claims;
- fake funding sources;
- unverified political statements;
- private patient information;
- internal system credentials;
- raw development notes;
- references suggesting AI-generated content.

## 4. Front-End Rules

- Prefer small, local changes.
- Avoid global CSS changes for local issues.
- Avoid rewriting full files unnecessarily.
- Preserve existing IDs, classes and anchors unless change is required.
- Preserve responsive behavior.
- Preserve accessibility improvements.
- Preserve SEO and Open Graph metadata.

## 5. Commit and Push

No commit or push may be performed without explicit authorization.

Commit authorization and push authorization are separate.
